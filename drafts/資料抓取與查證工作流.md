# 資料抓取與查證工作流（標準化作業）

> 本文件把來源報告附錄 B 的查證方法論與附錄 C 的來源分級，落地成儀表板每月更新的標準作業。指標定義、門檻與月度取值口徑以設計圖 §9 為準，本檔只規範「怎麼抓、怎麼驗」。API 系列代碼為建議值，首次實作時需上 FRED／來源網站複核後才鎖定。

## 1. 來源分級與處理原則（承附錄 C）

| 分級 | 代表來源 | 處理規則 |
|---|---|---|
| 一手／官方 | FINRA margin statistics、Fed H.4.1／FOMC 聲明與 SEP、財政部 QRA／TBAC、BoJ 決議文、公司財報與 SEC 文件、CBO | 直接採用 |
| 官方原始市場數據 | FRED 系列、Yahoo Finance 日線（自行下載）、Cboe 成交量新聞稿、ICI 週報、NY Fed ACM | 直接採用 |
| 二手（具名機構研究） | FactSet Earnings Insight、Citadel Securities 半年報、投行策略觀點、BIS／IMF、CoinGlass／Farside、GuruFocus／MacroMicro | 採用，但必須註明機構名與發布日期 |
| 三手（媒體轉述、彙整站） | Fortune、CNBC、Bloomberg 轉述、各彙整站 | 至少兩個獨立來源交叉比對後才採用；單源且口徑可疑者明文拒用並記錄拒用理由 |

通則：官方來源優先；同一數字有官方版本時，不得以二手／三手版本入庫。

## 2. 核心指標抓取分類

抓取方式定義：

- **自動**：有穩定 API 或可程式下載，腳本抓取後直接落地 CSV；每月仍需人工抽核 3 筆對照來源網站原始值。
- **半自動**：可程式抓取，但屬二手／彙整來源，需第二來源交叉比對，或抓下來的原始值需人工計算/判讀後才成為指標值。
- **人工**：無 API、發布為 PDF／新聞稿，或本質是質性判讀（語調標籤）。

| 指標 id | 來源（分級） | 抓取方式 | 建議管道 | 發布節奏與時點 |
|---|---|---|---|---|

輔助觀察指標（估值背景、市場廣度、AI 債務化、海外流動性）比照上表原則分類：有 FRED／Yahoo 系列者自動，需讀財報、發債公告、央行聲明者人工。

## 3. 月度工作流

### D1（每月第 1 個工作日）：自動批次

1. 跑自動抓取腳本：HY/IG OAS、BTC、SOFR-IORB、ON RRP、TGA、穩定幣供給、BTC ETF flow，落地當月 CSV（欄位依設計圖 §9）。
2. 抽核：每個自動來源抽 1 筆，對照來源網站原始值（附錄 B 的自行下載驗證精神）。
3. 半自動項交叉比對：穩定幣月末值對第二來源；差異 >2% 時兩值並列、標註口徑差異，不擇一隱藏。
4. 依門檻規則產生各指標 signal_state；FINRA、EPS 等未到者標 stale/missing，全局分數依規則標「暫估」。

### FINRA 發布後（約當月下旬）：人工補件與正式計分

1. 自 FINRA 頁面人工下載融資餘額，計算連續兩月變化，入庫。
2. 摘錄本週 FactSet Earnings Insight 的 NDX forward EPS 與修正方向。
3. 重算全局分數與三層＋AI 盈餘橋分數；覆蓋率達標後移除「暫估」標記。
4. 產出月報（設計圖 §11 格式）。

### 事件驅動（FOMC、NVIDIA/hyperscaler 財報、重大發債）

- 會後／發布後 2 個工作日內完成質性標註（fed_path 語調、EPS 指引方向），寫入 analyst_note。
- 事件註記不覆蓋月度主結論（設計圖 §2 既有規則）。

## 4. 查證紀律（承附錄 B，每月發布前逐項勾選）

- [ ] 資料截止日紀律：本月所有數值的 `data_as_of` 不晚於月報宣告的截止日；截止日後的資料留待下月。
- [ ] 逐條附源：每筆入庫值都有 `source_name`＋日期；二手來源附機構名與發布日。
- [ ] 官方優先：同一數字存在官方版本時，未使用二手／三手替代。
- [ ] 三手交叉：所有三手來源都有兩源比對紀錄；拒用的單源數字記錄拒用理由。
- [ ] 查不到不編：抓不到的值標 missing，並在 analyst_note 記錄試過的來源與搜尋（「我不知道」清單制）；嚴禁以推估值冒充實際值。
- [ ] 價格複核：凡引用自媒體／報告的價格或指數水位，以 Yahoo Finance 自行下載複核一次。
- [ ] 沿用揭露：沿用上月值者一律標 stale 並註明沿用（設計圖 §9 規則）。
- [ ] 對抗式審查（選用）：季度末或結論轉向的月份，將當月判讀交無寫作脈絡的第二方審查，重點查「否證門檻是否被迴避」。

## 5. 對資料模型的擴充建議

`indicator_definitions`（設計圖 §8.1）建議增加兩個欄位，讓抓取方式可稽核：

```json
"source_tier": "official | official_market_data | secondary | tertiary",
"fetch_mode": "auto | semi_auto | manual"
```

## 6. 開放事項
